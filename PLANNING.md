## Known Bugs (Fixed This Session)

### Thoughtseize P2 Bug (FIXED — commit 42c2cc3)
**What**: `best_proactive_target(gs)` always searched `gs.p2.hand`. When a Thoughtseize
deck was P2, it searched its own hand → `opponent.hand.remove(target)` crashed → turn forfeited.
**Impact**: 50% of Dimir games had strategy errors. 5 decks affected (dimir, bug, dimir_b,
mono_black, mardu). Biggest swing: Burn vs Dimir_b 88% → 69% (-19pp).
**Fix**: `best_proactive_target(gs, opponent=None)` now accepts explicit opponent parameter.
**Status**: Fixed and pushed. Matrix needs re-run.

### pills() Missing from Matrix HTML (FIXED)
**What**: `pills(cards, color)` function wasn't defined, causing ReferenceError on matchup click.
**Fix**: Injected before `closeDet()`. Now in CLAUDE.md Never list.

### classify_play 'other' Category (FIXED)
**What**: 11 play types were falling through to 'other' (SSG, Waste→, KozCmd, bare creature names).
**Fix**: Expanded to 17 categories with abbreviation handling. Fallback = 'spell', never 'other'.

## Architecture Reminders

### Data Pipeline
```
run_game() × 50/pair → meta_fresh.json (D)
                      → deck_agg.json (DA)
extract_cards.py      → card_trimmed.json (C)
extract_interactions  → interact_v3.json (I)
DECKS + agg           → ARCH map
```

### API Gotchas
- `run_game(d1, d2)` — d1 is always P1. Returns GameResult.
- `p1_opening_hand` is **list of strings**, not Card objects. Don't call `.name`.
- `best_proactive_target(gs, opponent)` — always pass opponent explicitly.
- Matchup keys in JSON use pipe separator: `d1|d2`.
- `game_length` counts rounds (both players), display turns are 2× game_length - 1.

- `cards.py DECKS dict`: 36 decks (no dimir_c, dimir_d — these use dimir base + different strategy in sim.py)
- `meta_fresh.json`: 36 decks in matrix
